Paisabazaar Banking Fraud Analysis - EDA
This repository contains a comprehensive Exploratory Data Analysis (EDA) focused on identifying key financial and behavioral factors that influence credit scores. Using a dataset of 100,000 customer records, this project provides actionable insights into risk assessment and creditworthiness.

📌 Project Overview
The objective is to help financial platforms like Paisabazaar improve loan approval processes and fraud detection. By analyzing variables such as income, debt, and payment history, we uncover the hidden patterns that define a "Good" vs. a "Poor" borrower.

📊 Key Insights
Behavior is King: Payment history and the number of delayed payments are the strongest predictors of a credit score—more than annual income.

The $1,500 Debt Ceiling: Regardless of income, outstanding debt exceeding $1,500 consistently triggers a "Poor" credit classification.

Market Opportunity: Data reveals a significant decline in users aged 50+, representing an underserved market for stable, long-term banking products.

Utilization Consistency: Credit utilization remains relatively uniform (around 32%) across all score tiers, suggesting it is a secondary risk factor in this specific dataset.

🛠️ Tech Stack
Language: Python

Libraries: Pandas, NumPy, Matplotlib, Seaborn, Plotly Express

Environment: Google Colab / Jupyter Notebook

📈 Analysis Workflow
Univariate Analysis: Examining distributions of Age, Income, and Credit Scores to establish a baseline.

Bivariate Analysis: Investigating relationships between variables (e.g., Debt vs. Credit Score).

Multivariate Analysis: Exploring complex interactions between income, utilization, and delays using heatmaps and 4D scatter plots.

Business Recommendations: Strategies for risk mitigation and product targeting based on data-driven thresholds.
